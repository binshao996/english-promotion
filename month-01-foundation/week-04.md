## 1. Grammar: Four Ways to Express Future

英语有四种完全不同的结构来表达将来。选哪一种取决于你对这件事的"确定程度"和"计划程度"。

---

### Part A: will（将 / 会 — 当场决定 / 预测 / 承诺）

**形式：** will + 动词原形（否定：will not = won't）

- **当场决定：** 说话时才决定做某事。
  - "The server just crashed — I'll check the logs."
  - "That's a good point. I'll add error handling."
- **主观预测：** 基于经验或直觉，不是具体证据。
  - "I think this architecture will scale to 10x traffic."
  - "The migration will probably take four hours."
- **承诺/提议：** 主动保证做某事。
  - "I'll send you the meeting notes after the call."
  - "Don't worry — I'll handle the deployment tonight."
- **客观事实：** 一定会发生的未来事件。
  - "The certificate will expire on June 30th."
  - "The database will reach its limit in two weeks."

---

### Part B: be going to（打算 / 将要 — 计划 / 有证据的预测）

**形式：** am/is/are + going to + 动词原形

- **预先计划：** 在说话前已经决定好的事。
  - "We're going to migrate the database this weekend."
  - "I'm going to refactor the auth module next sprint."
- **有证据的预测：** 根据当前迹象推断。
  - "Look at these error rates — the server is going to crash."
  - "The disk usage is at 95%. It's going to run out of space."
- **常见错误：** 忘记 be 动词。
  - ❌ "We going to deploy tonight."
  - ✅ "We **are** going to deploy tonight."

---

### Part C: Present Continuous for Future（现在进行时表将来 — 已安排的日程）

**形式：** am/is/are + V-ing + 将来时间词

- **已安排好的约会/会议/行程。** 必须加将来时间词！
  - "I'm meeting the product manager tomorrow at 3 PM."
  - "We're deploying the security patch tonight at midnight."
  - "He's flying to Singapore on Monday for the conference."
- **关键规则：** 没有将来时间词，听起来就像"现在正在做"。
  - ❌ "I'm meeting the client."（是现在还是将来？不清楚）
  - ✅ "I'm meeting the client **tomorrow**."
- **不能用于自然现象：** ❌ "It's raining tomorrow." ✅ "It's going to rain tomorrow."

---

### Part D: Present Simple for Future（一般现在时表将来 — 固定时间表）

**形式：** V / V-s/es + 将来时间

- **仅用于固定时间表：** sprint 截止、会议起止、列车时刻等"板上钉钉"的事。
  - "The sprint ends on Friday, so we need to wrap up."
  - "The deployment window opens at 10 PM and closes at 2 AM."
  - "The new API version goes live next quarter."
- **常见动词：** start, end, begin, finish, open, close, arrive, depart, go live
- ❌ "The sprint will end on Friday."（语法对但不自然——sprint 时间是固定的）
- ✅ "The sprint **ends** on Friday."（时间表用一般现在时更地道）

---

### Part E: Quick Contrast Table

| 表达方式 | 核心含义 | 何时使用 | 一句话对比 |
|---------|---------|---------|-----------|
| **will** | 当场决定/预测/承诺 | 说话时才做的决定；基于主观的预测 | "I'll look into it."（当场揽活） |
| **be going to** | 计划/预兆 | 提前计划好；有明显证据的预测 | "I'm going to fix it after lunch."（计划好的） |
| **Present Continuous** | 安排好的日程 | 有具体时间的约会、会议 | "I'm debugging it with Sarah at 3 PM."（日程安排） |
| **Present Simple** | 固定时间表 | 官方起止时间，不受个人影响 | "The sprint ends on Friday."（时间表固定） |

### Part F: Complex Sentence Breakdowns（复杂句拆解）

以下是 6 个包含将来表达的技术英语复杂句，逐词拆解分析。

---

#### 句子 1: If we don't fix this race condition before the next release, we are going to face a production outage that will affect millions of users who rely on our payment API.

```
分析:
- If (连词/条件状语从句引导) → 引导条件状语从句
- we (主语/代词) + don't fix (谓语/否定一般现在时) + this race condition (宾语/名词短语)
  + before the next release (时间状语/介词短语)
- , (逗号分隔主从句)
- we (主语/代词) + are going to (表将来/be going to) + face (谓语/动词原形)
  + a production outage (宾语/名词短语)
- that (关系代词/定语从句引导) + will affect (谓语/will + 原形)
  + millions of users (宾语/名词短语)
- who (关系代词/定语从句引导) + rely on (谓语/动词短语) + our payment API (宾语/名词短语)

说明:
- 这是一个"条件状语从句 + 主句 + 定语从句"的嵌套复合句。
- 条件从句 "If we don't fix..." 用一般现在时（don't fix），不能用将来时。
- 主句 "we are going to face" 用 be going to 表示"基于当前情况的预测"——
  如果不修复竞态条件→必然会面临 outage。
- "that will affect..." 是定语从句修饰 outage，用 will 因为这是预测结果。
- "who rely on..." 是定语从句修饰 users，用一般现在时表示"常规依赖"。
- 嵌套结构：条件从句(If...) → 主句(we are going to face...) → 定语从句1(that will affect...) → 定语从句2(who rely on...)

常见错误:
❌ "If we **won't fix** this race condition..."（条件从句中不能用 will）
✅ "If we **don't fix** this race condition..."（用一般现在时表将来条件）
```

---

#### 句子 2: By the time the security audit is complete, we will have already deployed the hotfix that patches the critical vulnerability which the penetration test uncovered last week.

```
分析:
- By the time (连词/时间状语从句引导) → "到……的时候"
- the security audit (主语/名词短语) + is (系动词) + complete (主语补足语/形容词)
- , (逗号)
- we (主语/代词) + will have already deployed (谓语/将来完成时)
  + the hotfix (宾语/名词短语)
- that (关系代词/定语从句引导) + patches (谓语/一般现在时)
  + the critical vulnerability (宾语/名词短语)
- which (关系代词/定语从句引导) + the penetration test (主语/名词短语)
  + uncovered (谓语/一般过去时) + last week (时间状语)

说明:
- "By the time + 现在时从句"表达"到将来某个时间点之前"。
- 时间从句 "the security audit is complete" 用一般现在时（is）表将来。
- 主句 "will have already deployed" 是将来完成时（will have + 过去分词）——
  强调在审计完成时，部署"已经完成了"。
- "that patches..." 是定语从句修饰 hotfix，用一般现在时描述补丁的功能。
- "which the penetration test uncovered..." 是嵌套定语从句修饰 vulnerability，
  用一般过去时因为漏洞发现是上周发生的。
- 三层嵌套：时间从句(By the time...) → 主句(we will have deployed...) → 
  定语从句1(that patches...) → 定语从句2(which...uncovered)

常见错误:
❌ "By the time the security audit **will be** complete..."（时间从句不能用将来时）
✅ "By the time the security audit **is** complete..."（用一般现在时）
```

---

#### 句子 3: The new monitoring system is going to alert us whenever a service exceeds its error budget, which means we will catch issues before they cascade into full-blown incidents.

```
分析:
- The new monitoring system (主语/名词短语) + is going to (表将来/be going to)
  + alert (谓语/动词原形) + us (间接宾语/代词)
- whenever (连词/时间状语从句引导) + a service (主语/名词短语)
  + exceeds (谓语/一般现在时) + its error budget (宾语/名词短语)
- , (逗号)
- which (关系代词/非限定性定语从句引导) + means (谓语/一般现在时)
- (省略 that 的宾语从句) we (主语/代词) + will catch (谓语/will + 原形)
  + issues (宾语/名词短语)
- before (连词/时间状语从句引导) + they (主语/代词) + cascade (谓语/一般现在时)
  + into full-blown incidents (介词短语/方式状语)

说明:
- "is going to alert" 用 be going to 表达"系统设计的功能"——这是系统的预期行为，
  不是临时决定，也不是固定时间表。
- "whenever" 引导时间状语从句，用一般现在时（exceeds）表将来条件。
- "which" 引导非限定性定语从句，指代前面整个句子（is going to alert us...）。
- "we will catch" 用 will 表达预测——"如果我们被提醒，我们就会抓到问题"。
- "before they cascade..." 用一般现在时（cascade）表将来，在 before 从句中。

常见错误:
❌ "...whenever a service **will exceed** its error budget"（时间从句不用 will）
✅ "...whenever a service **exceeds** its error budget"（一般现在时）
```

---

#### 句子 4: We are presenting the new architecture to the CTO tomorrow, and if she approves it, we will start the migration next sprint and complete it within six weeks.

```
分析:
- We (主语/代词) + are presenting (谓语/现在进行时表将来)
  + the new architecture (宾语/名词短语) + to the CTO (间接宾语/介词短语)
  + tomorrow (时间状语)
- , (逗号) + and (并列连词)
- if (连词/条件状语从句引导) + she (主语/代词) + approves (谓语/一般现在时)
  + it (宾语/代词)
- , (逗号)
- we (主语/代词) + will start (谓语/will + 原形) + the migration (宾语/名词短语)
  + next sprint (时间状语) + and (并列连词)
- complete (谓语/动词原形, 与 will start 并列) + it (宾语/代词)
  + within six weeks (时间状语/介词短语)

说明:
- "are presenting...tomorrow" 用现在进行时表将来，因为"明天向 CTO 演示"是已经安排好的日程。
- "if she approves it" 条件从句用一般现在时（approves），不用将来时。
- "we will start...and complete" 主句用 will，表达"如果条件满足，就……"的结果/承诺。
- "complete" 承前省略了 will，是 "will start...and (will) complete" 的省略形式。
- 并列结构：主句(we are presenting) + and + 条件从句(if...) + 主句(we will start...and complete...)
- 不同将来表达共存：现在进行时(已安排) + will(条件结果)，在同一句话中自然切换。

常见错误:
❌ "if she **will approve** it"（条件从句中不用 will）
✅ "if she **approves** it"（用一般现在时）
```

---

#### 句子 5: The flaky test has been causing random CI failures for weeks, but nobody has had the bandwidth to investigate it thoroughly because the team is preparing for the upcoming compliance audit.

```
分析:
- The flaky test (主语/名词短语) + has been causing (谓语/现在完成进行时)
  + random CI failures (宾语/名词短语) + for weeks (时间状语/介词短语)
- , (逗号) + but (并列连词/转折)
- nobody (主语/代词) + has had (谓语/现在完成时) + the bandwidth (宾语/名词短语)
  + to investigate it thoroughly (不定式短语/目的状语)
- because (连词/原因状语从句引导)
- the team (主语/名词短语) + is preparing (谓语/现在进行时)
  + for the upcoming compliance audit (目的状语/介词短语)

说明:
- "has been causing" 是现在完成进行时——描述"从过去持续到现在、仍在继续"的动作。
  这个 flaky test 过去几周一直在导致 CI 失败，而且可能还在继续。
- "has had" 是现在完成时——描述"从过去到现在"的时间段内没人有 bandwidth。
- "for weeks" 是现在完成时/完成进行时的标志性时间短语。
- "is preparing..." 用现在进行时描述团队"正在进行的准备工作"。
- "the upcoming compliance audit" 中的 "upcoming"（即将到来的）用形容词表达将来，
  没有使用将来时态。
- 虽然这句话没有使用任何将来时结构，但它自然地融入了"将来"的概念（upcoming audit）。

常见错误:
❌ "The flaky test **is causing** random CI failures for weeks"（for weeks 要求完成时）
✅ "The flaky test **has been causing** random CI failures for weeks"（完成进行时表持续）
```

---

#### 句子 6: Once we finish refactoring the authentication module, the entire system is going to be more secure, easier to maintain, and significantly faster to deploy.

```
分析:
- Once (连词/时间状语从句引导) → "一旦……就……"
- we (主语/代词) + finish (谓语/一般现在时) + refactoring (动名词/宾语)
  + the authentication module (宾语/名词短语)
- , (逗号)
- the entire system (主语/名词短语) + is going to be (谓语/be going to + 原形)
  + more secure (主语补足语/形容词比较级)
  + , (逗号) + easier to maintain (并列主语补足语/形容词比较级 + 不定式)
  + , (逗号) + and (并列连词)
  + significantly faster to deploy (并列主语补足语/副词 + 形容词比较级 + 不定式)

说明:
- "Once we finish refactoring..." 时间状语从句用一般现在时（finish），不用将来时。
- "refactoring" 是动名词作 finish 的宾语——固定搭配 "finish + V-ing"。
- "is going to be" 用 be going to 表达"基于计划的预测"——重构完成后，系统自然会变好。
- 三个并列的补足语（more secure, easier to maintain, faster to deploy）用逗号 + and 连接，
  这是英语中三项并列的标准写法——逗号分隔前两项，and 连接最后一项（Oxford comma 可选）。
- "significantly" 是副词修饰比较级 "faster"。

常见错误:
❌ "Once we **will finish** refactoring..."（时间从句不用 will）
✅ "Once we **finish** refactoring..."（用一般现在时）
❌ "Once we finish **to refactor**..."（finish 后接动名词，不是不定式）
✅ "Once we finish **refactoring**..."（动名词作宾语）
```

## 2. Month 1 Grammar Review

### A. Tense Overview Table（时态概览表）

Month 1 学过的全部 8 种时态/语法结构的总览：

| 时态 | 公式 | 核心用法 | 一句话示例 |
|------|------|---------|-----------|
| **Simple Present** 一般现在时 | V / V-s/es | 事实、习惯、当前状态 | "We deploy every Monday." |
| **Present Continuous** 现在进行时 | am/is/are + V-ing | 正在进行、临时情况 | "I'm debugging a memory leak." |
| **Simple Past** 一般过去时 | V-ed / 不规则过去式 | 过去完成的动作 | "The server crashed at 2 AM." |
| **Past Continuous** 过去进行时 | was/were + V-ing | 过去某时刻正在进行的动作 | "I was reviewing the PR when you called." |
| **will + 原形** 一般将来时 | will + V | 当场决定、预测、承诺 | "I'll check the logs." |
| **be going to** 打算将来 | am/is/are + going to + V | 计划、有证据的预测 | "We're going to migrate the DB." |
| **Present Continuous (将来)** | am/is/are + V-ing + 将来时间 | 已安排的日程 | "I'm meeting the PM tomorrow." |
| **Present Simple (将来)** | V-s/es + 将来时间 | 固定时间表 | "The sprint ends on Friday." |

**时态选择核心原则：** 时间（过去/现在/将来）+ 状态（简单/进行）= 正确时态。

---

### B. Sentence Patterns Review（5 种句型回顾）

**Pattern 1: S + V（主语 + 不及物动词）**
- "The production server crashed last night."
- The production server (主语) + crashed (谓语/不及物动词) + last night (时间状语)
- crashed 是不及物动词，后面不能加宾语。

**Pattern 2: S + V + O（主语 + 及物动词 + 宾语）**
- "We patched the security vulnerability yesterday."
- We (主语) + patched (谓语/及物动词) + the security vulnerability (宾语) + yesterday (时间状语)
- patched 是及物动词，动作作用在 the security vulnerability 上。

**Pattern 3: S + V + C（主语 + 系动词 + 补足语）**
- "The flaky test is a major blocker for the release."
- The flaky test (主语) + is (系动词) + a major blocker (补足语) + for the release (介词短语)
- 系动词 is 连接主语和补足语，相当于等号。

**Pattern 4: S + V + O + O（主语 + 动词 + 间接宾语 + 直接宾语）**
- "The team lead assigned me the race condition investigation."
- The team lead (主语) + assigned (谓语) + me (间接宾语/人) + the race condition investigation (直接宾语/事)
- 间接宾语是人（me），直接宾语是事（the investigation）。人→物。

**Pattern 5: S + V + O + C（主语 + 动词 + 宾语 + 宾语补足语）**
- "The outage kept the service unavailable for two hours."
- The outage (主语) + kept (谓语) + the service (宾语) + unavailable (宾语补足语) + two hours (时间状语)
- unavailable 描述宾语 the service 的状态——"服务处于不可用状态"。

### C. Top 10 Most Common Mistakes（Month 1 常见错误 TOP 10）

**1. 忘记第三人称单数 -s（Simple Present）**
- ❌ "The API return a JSON object."
- ✅ "The API return**s** a JSON object."

**2. 现在进行时忘记 be 动词**
- ❌ "I debugging the issue."
- ✅ "I **am** debugging the issue."

**3. 状态动词用了进行时**
- ❌ "I am knowing the answer."
- ✅ "I know the answer."

**4. 双重过去标记（did + 过去式）**
- ❌ "I didn't **deployed** the fix."
- ✅ "I didn't **deploy** the fix."

**5. 不规则动词用成规则变化**
- ❌ "I **writed** a script."
- ✅ "I **wrote** a script."

**6. 所有将来都用 will**
- ❌ "I **will meet** the client tomorrow at 3."（听起来像当场决定）
- ✅ "I'm meeting the client tomorrow at 3."（安排好用现在进行时）

**7. will 后面加 -s 或 -ing**
- ❌ "It **will works**." / "It **will working**."
- ✅ "It **will work**."

**8. 时间/条件状语从句中用 will**
- ❌ "I'll deploy when the tests **will pass**."
- ✅ "I'll deploy when the tests **pass**."

**9. be going to 忘记 be 动词**
- ❌ "We **going to** migrate the database."
- ✅ "We **are going to** migrate the database."

**10. 同一段话中时态前后不一致**
- ❌ "Yesterday, I **deploy** the fix. The server **crashes** and I **fix** it."
- ✅ "Yesterday, I **deployed** the fix. The server **crashed** and I **fixed** it."

## 3. Vocabulary: 300 B2-C1 Core Words

### Group 1: Quality, Reliability & Security Terms（50 词）

### 1. integrity
- **义项 1: 完整性 (n.)** — 数据未被篡改或损坏的状态
  - "The database **integrity** check verified that no records were corrupted during the unexpected shutdown."
- **义项 2: 诚信 (n.)** — 道德上的正直和诚实
  - "A security engineer must demonstrate professional **integrity** when handling sensitive customer data."
- **义项 3: 统一性 (n.)** — 系统各部分协调一致的状态
  - "The referential **integrity** of the database ensures that foreign key relationships remain valid."

### 2. confidentiality
- **义项 1: 机密性 (n.)** — 信息仅对授权者可见的特性
  - "The encryption protocol guarantees **confidentiality** by ensuring only the intended recipient can read the message."
- **义项 2: 保密义务 (n.)** — 不泄露秘密的法律或道德责任
  - "All employees must sign a **confidentiality** agreement before accessing the source code repository."
- **义项 3: 隐私保护 (n.)** — 个人信息不被公开的状态
  - "The **confidentiality** of patient medical records is protected under strict regulatory frameworks."

### 3. availability
- **义项 1: 可用性 (n.)** — 系统可被访问和使用的状态
  - "Our SLA guarantees 99.99% **availability**, which translates to less than one hour of downtime per year."
- **义项 2: 可获得性 (n.)** — 资源或信息可以被获取的状态
  - "The **availability** of skilled Kubernetes engineers has become a bottleneck for our infrastructure expansion."
- **义项 3: 有效性 (n.)** — 某物存在并可被使用的状态
  - "The **availability** of real-time monitoring data allows us to detect anomalies within seconds."

### 4. authenticity

### 201. explicit

  - "The team has **implicit** trust in the automated testing pipeline based on its track record."

### 251. boil down to
- **义项 1: 归结为 (v.)** — 某复杂事物的最核心或最根本的原因
  - "The performance issue **boils down to** a single inefficient database query that touches too many rows."

  - "The 50-page incident report **boils down to** three root causes and five action items."
- **义项 3: 本质上就是 (v.)** — 排除细节后的核心事实
  - "All architectural decisions **boil down to** trade-offs between competing constraints."

### 252. fall through
- **义项 1: 落空/失败 (v.)** — 计划或安排未能实现
  - "The migration plan **fell through** when the vendor withdrew their support for the legacy system."
- **义项 2: 穿过 (v.)** — 从缝隙或漏洞中掉落
  - "The bug **fell through** the cracks in the testing process and made it all the way to production."
- **义项 3: 透过 (v.)** — 物理上从某层掉落
  - "The request **falls through** to the default handler when none of the specific conditions match."

### 253. pan out
- **义项 1: 成功/有结果 (v.)** — 事情按预期发展并取得好的结果
  - "The experimental caching strategy **panned out** — response times dropped by 60%."
- **义项 2: 结果如何 (v.)** — 某事最终的结果（常用于否定句）
  - "The first approach didn't **pan out**, so the team pivoted to an alternative solution."
- **义项 3: 出成果 (v.)** — 经过努力产生积极结果
  - "We invested three months in the prototype, and fortunately, it **panned out**."

### 254. rule out
- **义项 1: 排除 (v.)** — 确定某可能性不成立
  - "The investigation **ruled out** a hardware failure after running diagnostic tests on all components."
- **义项 2: 不考虑 (v.)** — 因不可行而排除某选项
  - "The budget constraints **rule out** the option of migrating to a new database this quarter."
- **义项 3: 阻止 (v.)** — 使某事不可能发生
  - "The security policy **rules out** the use of any unencrypted communication protocols."

### 255. iron out
- **义项 1: 解决/消除 (v.)** — 解决小问题或消除困难
  - "The team spent the last week of the sprint **ironing out** the remaining edge cases in the payment flow."
- **义项 2: 理顺 (v.)** — 解决分歧或误解
  - "We need to **iron out** the discrepancies between the staging and production configurations."
- **义项 3: 完善 (v.)** — 使某事变得顺利和完善
  - "The deployment process still has some rough edges that need to be **ironed out**."

### 256. hash out
- **义项 1: 详细讨论 (v.)** — 深入讨论以达成共识或解决方案
  - "The architects spent two hours **hashing out** the details of the new authentication protocol."

  - "The team **hashed out** their differences about the testing strategy during the retrospective."
- **义项 3: 讨论敲定 (v.)** — 通过交谈完善某计划或想法

- **义项 2: 相当于 (v.)** — 在某效果或意义上等同于
  - "All the small optimizations together **amount to** a 30% improvement in overall system performance."
- **义项 3: 有所成就 (v.)** — 发展成有价值的成果（常用于否定）
  - "The ambitious project ultimately **amounted to** nothing because the requirements kept changing."

## 4. Sentence-Making Practice

结合 Month 1 全部语法点和本周 300 个 B2-C1 核心词汇的 10 个实践练习。

---

### 1. 你在 standup 上说：安全审计揭示了认证服务中的一个漏洞，你打算今天下午调查。用 **一般过去时** + **be going to**，使用 **vulnerability**（Group 1）。

**参考答案:** "The security audit revealed a vulnerability in the authentication service. I'm going to investigate it this afternoon."
> 分析: revealed 是一般过去时。I'm going to investigate 是 be going to 表计划。"reveal a vulnerability" 是"security"领域的常见搭配。

---

### 2. 你告诉同事：一旦我们敲定了迁移计划的具体细节，我们就开始执行。用 **时间状语从句（一般现在时表将来）** + **will**，使用 **iron out** 或 **flesh out**（Group 6）。

**参考答案:** "Once we flesh out the details of the migration plan, we will kick off the execution."
> 分析: Once we flesh out 是时间状语从句，用一般现在时表将来。will kick off 是主句将来时。flesh out 和 kick off 都是本周学过的习语。

---

### 3. 你描述昨晚的 incident：你在查看监控仪表盘的时候，CPU 利用率突然飙升到了 95%。用 **过去进行时 + when + 一般过去时**，使用 **spike**（Group 2）。

**参考答案:** "I was observing the monitoring dashboard when CPU utilization spiked to 95%."
> 分析: was observing 是过去进行时（背景动作），spiked 是一般过去时（打断事件）。utilization 表达"利用率"，spike 作动词表示"突然飙升"。

---

### 4. 你向团队建议：如果你发现了数据损坏，应该立即上报给 SRE 团队。用 **条件状语从句（一般现在时）** + **should**，使用 **corrupt**（Group 1）。

**参考答案:** "If you detect corrupt data, you should escalate it to the SRE team immediately."
> 分析: If you detect... 条件从句用一般现在时。should escalate 表示"应该上报"。detect 和 escalate 都是学术/技术场景中的高频动词。

---

### 5. 你告诉朋友你周末的安排：周六早上和同事参加安全研讨会，下午写代码。用 **现在进行时表将来**，使用 **colleague**、**workshop**（Group 3/4）。

**参考答案:** "I'm attending a security workshop with my colleagues on Saturday morning, and I'm reviewing pull requests in the afternoon."
> 分析: 'm attending 和 'm reviewing 都是现在进行时表将来（已安排的活动）。必须加时间词（Saturday morning / in the afternoon）。

---

### 6. 你解释为什么 Sprint 周五结束（固定时间表），而且你们进度落后了。用 **一般现在时表将来** + **be going to**，使用 **fall behind**（Group 6）。

**参考答案:** "The sprint ends on Friday, and we're going to fall behind on the deadline if we don't speed up."
> 分析: ends 是一般现在时表固定时间表。're going to fall behind 是 be going to 表预测。if we don't speed up 是条件从句。

---

### 7. 你在面试中描述你过去解决过的一个数据库死锁问题，以及这次经历如何让你理解了数据一致性的重要性。用 **一般过去时**，使用 **deadlock**、**consistency**（Group 1/2）。

**参考答案:** "I resolved a database deadlock last year, and the experience taught me the importance of data consistency."
> 分析: resolved 和 taught 都是一般过去时。taught 是 teach 的不规则过去式。consistency 是 Week 4 新增的精确术语。

---

### 8. 你在规划会议上说：团队已经连续两个 sprint 滞后了，我们需要找出瓶颈在哪里。用 **现在完成时**，使用 **fall behind**、**identify**（Group 4/6）。

**参考答案:** "The team has fallen behind for two consecutive sprints. We need to identify where the bottleneck is."
> 分析: has fallen 是现在完成时（fall→fallen→fallen）。for two consecutive sprints 表示持续时间。identify 表示"找出/识别"。

---

### 9. 你向新同事解释：我们通常周二部署，但下周我们周四部署，因为团队周三要参加一个合规审计。用 **一般现在时（常规）** + **be going to（计划）**，使用 **compliance**、**audit**（Group 1）。

**参考答案:** "We usually deploy on Tuesdays, but we're going to deploy on Thursday next week because the team is attending a compliance audit on Wednesday."
> 分析: deploy 是一般现在时（常规操作）。're going to deploy 是计划。is attending 是现在进行时表将来（已安排的日程）。

---

### 10. 你在写 incident postmortem：这次安全漏洞是因为我们跳过了安全审查——这个教训我们不会再忘记。用 **一般过去时** + **won't**，使用 **breach**、**review**（Group 1）。

**参考答案:** "The breach happened because we skipped the security review. We won't forget this lesson."
> 分析: happened 和 skipped 都是一般过去时。won't forget 是 will 的否定将来，表达"坚定不会忘记"。breach 在这里指"安全漏洞/入侵事件"。

## 6. Weekend Review

### A. Commonly Confused Word Pairs（Week 4 精选 10 对）

**1. explicit vs. implicit（明确的 vs. 隐含的）**
- "The error message provides **explicit** instructions for resolving the conflict."
- "There is an **implicit** assumption that the database will always be available."

**2. tangible vs. intangible（有形的 vs. 无形的）**
- "The performance optimization produced **tangible** results — a 40% improvement."
- "The **intangible** benefits of good documentation include faster onboarding."

**3. proactive vs. reactive（主动的 vs. 反应式的）**
- "A **proactive** monitoring strategy detects issues before they impact users."
- "A **reactive** approach only fixes problems after they've caused damage."

**4. deterministic vs. stochastic（确定性的 vs. 随机的）**
- "**Deterministic** algorithms always produce the same output for a given input."
- "Network latency is inherently **stochastic** — it always exhibits variation."

**5. empirical vs. theoretical（经验的 vs. 理论的）**
- "**Empirical** data from load testing revealed patterns the models never predicted."
- "The **theoretical** maximum throughput is 50,000 requests per second."

**6. objective vs. subjective（客观的 vs. 主观的）**
- "Benchmark results provide **objective** data for architectural decisions."
- "Code style preferences are largely **subjective**."

**7. centralized vs. decentralized（集中的 vs. 分散的）**
- "A **centralized** logging system collects logs from all services."
- "A **decentralized** architecture empowers individual teams to make decisions."

**8. abstract vs. concrete（抽象的 vs. 具体的）**
- "The architecture diagram is an **abstract** representation of the system."
- "The postmortem should include **concrete** action items with assigned owners."

**9. correlate vs. attribute（关联 vs. 归因）**
- "We **correlated** the spike in errors with the new deployment."
- "Don't **attribute** the performance gain to the new algorithm without evidence."

**10. reveal vs. expose（揭示 vs. 暴露）**
- "The profiling **revealed** that 80% of the time was spent in one query."
- "The misconfigured bucket **exposed** customer records to the public."
